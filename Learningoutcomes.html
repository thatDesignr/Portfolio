New Marktplaats.nl UI/UX

In semester 2 I have made interactive media products that I have made with the program Figma. For example, I made a UI/UX overhaul of Marktplaats.nl

About the Marktplaats.nl UI/UX overhaul

The current Marktplaats UI/UX over the homepage looks old and has a bad UX. After doing research how other websites that sell goods and Marktplaats’ her competitors do, I have concluded a few things;

Many other websites have an advances search function, which the current Marktplaats side does not have. The current Marktplaats website shows notifications, even when not logged in, which does not make sense and is just confusing.

Besides those UX improvements, I also added the most sought/bought items at the top of the website with so users do not have to search for them.

I also improved the look of the UI, while staying in the same layout, so existing users are not confused by the changes.

Documentation

Documentation in web development serves as a crucial roadmap for developers, offering clear instructions, explanations, and examples on how to use, maintain, and extend code. It enhances collaboration among team members, ensures consistency in coding practices, and facilitates easier onboarding of new developers. Additionally, comprehensive documentation fosters better understanding of the project for stakeholders and clients, leading to improved communication and more effective decision-making.

Version Control History Environment

Version control history is important because it provides a detailed record of changes made to a codebase over time. It allows developers to track the evolution of a project, understand who made specific changes, and revert to previous versions if necessary. Additionally, version control history facilitates collaboration among team members by providing insights into the development process and ensuring consistency across code contributions.

Working together in Github

Because I did this project together with Levi and Hein, we worked in Github. Everyone had their own branch where we would pull, commit and merge. This was very challenging to do because it always produced conflicts and errors. In the end we learned alot from it about how to work together. For example what we did wrong was not make a general stylesheet for things that are always the same like text sizes, fonts, colors, etc. Also, everyone codes in a different style, for example some like to use sections instead of divs, some used percentages instead of pixels, and some use grids where others don't. This made the whole website a mess, but for next time we know that before we start coding we need to make agreements.

Identifying my own core values

Thinking about what's important to me in my studies and work, I've figured out a few things that really matter. I discovered these by looking at myself and my experiences.

One big moment was during my second semester as an ICT media & design student. I was doing projects that mixed design and front-end coding. Doing these projects made me realize how much I care about being creative, solving problems, and working well with others.

So, after some thinking, I found some values that are super important to me. Being creative is a big one. It drives me to think in new ways and make things that look good and work well. I also really value teamwork. I think we can do amazing things when we work together and support each other. And being honest and doing things well is really important too.

These values guide me in my studies, career, and work. In my studies, they push me to take on challenging projects with my classmates and make sure we all work together well. For example, when I work with three other students, I make sure we talk openly, respect each other, and all do our part.

For my career, these values help me choose jobs that match what I believe in and let me make a positive difference. Whether it's making cool designs, coming up with new tech ideas, or making sure things are done right, my values guide my choices.

When I work with clients, like teachers I've worked with before, I focus on understanding what they need, being honest, and doing my best work. By sticking to my values, I hope to not just do well, but also make make the client happy.
